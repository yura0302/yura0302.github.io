### 리액트란

- **페북에서 제공하는 프론트엔드 라이브러리로 UI를 만들기 위한 웹애플리케이션 뿐만아니라 네이티브 앱 등 다양한 플랫폼에서 활용되는 JS라이브러리이다.**
- 컴포넌트 UI를 재사용 가능한 작은 부분으로 나누고 자체적으로 작동할 수 있으며 다른 컴포넌트와 조합하여 더 큰 컴포넌트를 형성할 수 있다.
- 리액트는 대규모 애플리케이션에서 유지보수 및 확장성이 높은 코드를 작성할 수 있도록 돕는다.

### 리액트 특징

- **컴포넌트 기반 (Component-Baesd)**
  - UI를 하나의 큰 덩어리로 생각한다면 컴포넌트는 그 덩어리를 이루는 작은 요소들이라고 생각하면 된다.
    컴포넌트는 작은 요소로 쪼개져 있기 때문에 전체 코드를 파악하기가 상대적으로 쉽다.
  - 또한 작게 쪼개면 다른 화면에서도 사용할 수 있기때문에 재사용성이 높고, 같은 코드를 반복해서 입력할 필요가 없기 때문에 효율적이다.
- **Virtual Dom**
  - Vitual DOM 은 주로 SPA(Single Page Application)에서 적용되는 개념으로 하나의 큰 html을 Real Dom으로 칭하고 해당 DOM을 컴포넌트 단위로 쪼개서 추상화한 개념이다.
  - Real DOM에서 직접 처리하는게 아니라 Virtual Dom이 버퍼 역할을 해줌으로써 어떤 부분이 바뀔지 미리 계산, 수정 후에 수정된 해당 부분만 업데이트 해준다.
    정리하자면 V-DOM은 변경된 부분만 Real DOM에 반영하는 방식으로 작업을 수행한다.
- **Data Flow (단방향)**
  - 부모→자식 컴포넌트로 데이터를 전달 할 수 있으나 역으로는 데이터 전송이 불가능하기 때문에 부모의 데이터를 바꿔주기 위해 state를 이용한다.
  - 또한 리액트는 컴포넌트 단위로 업데이트를 수행하는데 단방향 데이터 흐름은 업데이트 과정을 더 효율적으로 만들어서 성능을 최적화하는데 컴포넌트간의 의존성이 낮아지기 때문에 업데이트를 수행 할 때 불필요한 랜더링을 방지할 수 있다.
- **JSX (JavaScript eXtension)**

  - React에서 HTML을 표현할 때 JSX를 사용하는데 외관상 HTML같은 마크업 언어를 리터럴로 입력하는것처럼 보이는데 빌드시 Babel에 의해 자바스크립트로 변환된다.
  - 자바스크립트 코드를 HTML처럼 표현할 수 있기 때문에 개발에 용이하다.

    cf. JSX vs HTML

    1. JSX의 표현식은 중괄호로 감싼다. { 표현식 }
    2. 만약 속성값이 문자열이면 따옴표로 감싼다.
    3. 만약 속성값이 표현식이면 중괄호로 감싼다.

- **유지보수 및 확장성 (Maintainability)**
  - 컴포넌트 기반 구조와 v-dom 등의 특징으로 유지보수와 확장에 용이하다
- **서버사이드 렌더링 (Server-Side-Rendering,SSR)**
  - 리액트는 기본적으로 CRS(Client Side Rendering) 을 지원하나 ssr도지원하여 csr의 단점이었던 초기 로딩 속도를 개선할 수 있다.
    cf. SSR을 위한 도구의 예시로 Next.js , Gatsby 등이 있다.
  - 또한 React에서는 ReactDOMServer 모듈을 제공하여 서버에서 컴포넌트를 렌더링 할 수 있도록 한다.
- **리액트 네이티브 (React Native) :** 웹 애플리케이션 뿐만 아니라 React Native를 이용하여 앱 개발도 가능하다.
