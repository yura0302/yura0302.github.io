title: AWS-엣지 연결

1. VPC에서 분리한 IGW 다시 연결
2. Pub-RTB 라우팅 테이블 선택 > 작업 > 라우팅 편집 > IGW 라우팅 삭제 후 변경 사항 저장
3. Pub-RTB 라우팅 테이블이 선택된 상태에서 엣지 연결 탭 > 엣지 연결 편집 클릭

   ![엣지 연결 오류 메세지 ](/assets/images/fail.png)

   엣지 연결 오류 메세지

   오류 발생 이유: 엣지 라우팅 대상을 VPC CIDR로 한정하지 않았기 때문이다.

   ENI 라우팅 대상 (13.246.100.25/32)를 92.75.20.0/24로 변경하면 Pub-RTB 라우팅 테이블은 서브넷 2개에 연결 돼 있으므로 VPC CIDR보다 작은 규모의 네트워크를 대상으로 등록할 수 없다. 따라서 **명시적으로 연결한 라우팅 테이블을 해제한 뒤 모든 서브넷이 기본 라우팅 테이블과 연결하도록 수정해야한다.**

4. 라우팅 테이블을 아래와같이 수정

   ![라우팅 대상을 서브넷으로 변경](/assets/images/half.png)

   라우팅 대상을 서브넷으로 변경

5. 실패한 엣지 연결을 다시 수행하면 라우팅 테이블이 IGW에 연결된 모습을 볼 수 있다.

   ![엣지 연결 결과 화면](/assets/images/result.png)

   엣지 연결 결과 화면

**정리**

- 라우팅 테이블에서 트래픽 목적지를 찾지 못하면 트래픽은 소멸된다.
- 인터넷을 타깃으로 지정한 기본 게이트웨이 라우팅은 반드시 필요할 때만 사용해야한다.
