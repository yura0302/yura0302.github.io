title: AWS-라우팅 테이블 생성 및 서브넷 연결
---


1. **서비스 > VPC > 라우팅 테이블 메뉴 > 라우팅 테이블 생성**
2. **이름 입력 후 라우팅 테이블 생성할 VPC 선택 > 라우팅 테이블 생성**

![creat-route.png](/assets/images/creat-route.png)

새로 생성한 테이블(Pub-RTB)와 기본 라우팅 테이블(rtb-06df~)가 보인다.

명시적 서브넷 연결 필드에 아무런 값도 없으므로 모두 기본 라우팅 테이블에 연결 돼 있음을 알 수 있다.

3. **서브넷 연결하기**

서브넷 연결 편집 > 퍼블릭 용도로 만든 서브넷 2개(PUB-WEB 2개) 를 선택 >연결 저장 클릭

명시적 서브넷 연결 필드에 ‘2 서브넷’표시

<br/>

**정리**

- 라우팅 테이블의 연결 대상은 서브넷이며 수명 주기동안 다른 서브넷에 동시적으로 연결 할 수 없음을 확인, 또한 어떤 서브넷에서도 연결하지 않은 상태로 존재할 수 없음을 알 수 있었다.

- 반대로 서브넷은 반드시 라우팅 테이블과 연결돼야하며 서브넷은 단 하나의 라우팅 테이블을 사용하지만 다른 라우팅 테이블로 바꿔 사용할 수 도있음을 알 수 있었다.
- 기본 VPC를 포함한 모든 VPC가 생성될 때 기본 라우팅 테이블도 함께 생성된다. 따라서 기본 라우팅 테이블의 개수와 VPC의 개수는 같다고 할 수 있다.
- VPC 내부에 서브넷을 생성하는 시점에 서브넷을 생성하면 일단 기본 라우팅 테이블에 자동 연결되있는 상태임을 알 수 있었다.
