### **Serverless란**

![Untitled](/assets/images/3232.png)

- **서버 관리와 프로비전 없이 코드를 실행**
- 사용한만큼 비용지불
- **고가용성 && 장애 내구성 (필요에 따라 자동으로 Scale up, Scale down)**
- 빠른 배포와 업데이트 가능

**→ 서버리스 환경에 맞는 인프라구축과 설계가 필요**

---

### **AWS Lambda란**

- 코드 실행을 위한 파일을 업로드하면 **서버 프로비전없이 코드를 실행**
- 다양한 AWS에서 람다를 활용
- 다양한 언어 지원
- Lambda 호출 방법
  - Event 기반
  - API Gateway를 통해 호출

---

### **AWS Lambda 구성**

1. **Deployment Package**
   - 함수 코드와 코드를 실행할 런타임으로 구성
   - S3업로드 가능
2. **Trigger (MSA에 적합)**
   - AWS를 호출하는 서비스
   - ex) API Gatewway,SQS,S3
   - 각 서비스에서 호출 시 지정된 양식의 이벤트 내용 전달
3. **권한 (IAM)**
4. **태그**
5. **환경변수**

---

### **테스트 이벤트**

- 람다 함수 테스트를 위한 내용을 템플릿으로 구성 가능
- 프라이빗
  - 함수 생성자만 작성 가능
- 공유가능
  - 같은 계정 다른 IAM 사용자에게 공유 가능
- 다양한 서비스에서 람다 호출시 전달하는 이벤트가 미리 지정되어있음

    1. **서버리스 컴퓨팅의 핵심 == aws Lambda 의 핵심 == Lambda함수**

    3. **aws에서 람다함수를 테스트 할 수 있는 기능을 제공**
    4. **테스트결과를 CloudWatch에서 함수 호출을 모니터 할 수 있음**
